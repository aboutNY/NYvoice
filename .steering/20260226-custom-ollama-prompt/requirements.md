# requirements.md

## 1. 目的
Ollama に渡す補正指示プロンプトを、固定実装からユーザー設定可能な仕様へ変更する。

## 2. 背景
現状は `OllamaCorrectionService` 内で補正プロンプトがハードコードされており、
利用者が文体・補正方針・厳密度を調整できない。

## 3. 要件
- 設定画面から Ollama 補正プロンプトを編集・保存できること
- 既定値は現行の補正方針（意味維持・要約禁止・修正テキストのみ返却）を維持すること
- プロンプト内で文字起こし結果を埋め込めること
- 既存ユーザー設定の互換性を壊さないこと

## 4. 非機能要件
- 既存の補正 ON/OFF およびモデル設定の動作を維持すること
- 空レスポンスなど既存のエラー/フォールバック動作を維持すること

## 5. 受け入れ基準
- 設定で保存したプロンプト内容が Ollama 呼び出し時に利用される
- プレースホルダー未記載時でも補正対象テキストがプロンプトに含まれる
- 既存保存データ読み込み時に設定が初期化されず、アプリが正常起動する
