# Glossary

## 1. 目的
本書は、NYvoiceプロジェクトで使用する用語の定義を統一し、設計・実装・レビュー時の認識ズレを防ぐための用語集である。

## 2. プロダクト用語
### 音声入力セッション
ユーザーが録音開始してから停止するまでの一連の処理単位。

### 録音モーダル
録音中に表示される小型UI。録音状態表示と停止操作を提供する。

### 補正
文字起こし結果に対してローカルLLMが実施する誤字・変換ミスの修正処理。

### カーソル位置挿入
現在フォーカスされている入力先に、最終テキストを挿入する操作。

### フォールバック
第一手段が失敗した際に、代替手段で目的を達成すること。例: 直接挿入失敗時のクリップボード経由。

## 3. 技術用語
### Whisper.cpp
ローカルで動作する音声認識エンジン。録音データを文字列へ変換する。

### Ollama
ローカルLLM実行ランタイム。文字起こし結果の文脈補正に利用する。

### Adapter
外部依存（Whisper.cpp、Ollama、OS APIなど）との接続をカプセル化する層。

### Session Controller
音声入力セッション全体の状態遷移を管理し、各サービスをオーケストレーションするコンポーネント。

### Domain Error
ドメイン上意味のある失敗分類。UI表示文言と技術詳細ログを分離して扱う。

### Global Shortcut
アプリ非アクティブ時でも有効なシステム全体ショートカット。

### Accessibility Permission
他アプリの入力領域への挿入を行うために必要なmacOS補助機能権限。

## 4. 状態管理用語
### Idle
待機状態。セッション未実行。

### Recording
録音中状態。音声を収集中。

### Transcribing
文字起こし処理中状態。Whisper.cpp実行中。

### Correcting
補正処理中状態。Ollama実行中。

### Inserting
テキスト挿入処理中状態。

### Error
いずれかの処理で失敗し、ユーザー通知または復帰処理を行う状態。

## 5. 配布・運用用語
### 単一パッケージ配布
チームメイトが1つのmacOSアプリ（.app）として受け取り実行できる配布形態。

### 初回導線
初回起動時に必要権限や依存環境を確認・案内する一連の画面フロー。

### 同梱資産
アプリ配布物に含める実行バイナリ、モデル、設定テンプレートなどの資産。

## 6. ドキュメント運用用語
### 永続ドキュメント
`docs/` 配下に置く、プロジェクト全体方針を示す文書。

### ステアリングファイル
`.steering/YYYYMMDD-task-name/` 配下で、作業単位の要求・設計・タスクを管理する文書群。

### 要件（requirements）
「何を満たすべきか」を定義する文書。

### 設計（design）
要件をどのように実現するかを示す文書。

### タスクリスト（tasklist）
実装・検証の実行手順と進捗を管理する文書。

## 7. 用語追加ルール
- 新しい専門語を導入した場合は `glossary.md` を同一PR内で更新する
- 略語を使う場合、初出時に正式名称を併記する
- 意味が曖昧な語は例を付けて定義する
