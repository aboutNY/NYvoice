# Functional Design

## 1. 目的
本書は、音声入力文字起こしアプリの機能仕様とユーザー操作フローを定義する。
対象はMVPとし、実装はSwiftを前提とする。

## 2. 画面・操作の全体像
本アプリはメニューバー常駐型で、主操作はグローバルショートカットで行う。
通常時は常駐のみで、録音中にモーダルを表示する。

### 2.1 状態一覧
- Idle: 待機中
- Recording: 録音中（モーダル表示）
- Transcribing: 文字起こし中
- Correcting: LLM補正中
- Inserting: テキスト挿入中
- Error: エラー表示中

## 3. ユーザーフロー
### 3.1 基本フロー
1. ユーザーがグローバルショートカットを押す
2. 録音を開始し、録音モーダルを表示する
3. ユーザーが話す
4. ユーザーが停止ボタンまたは同ショートカット再押下で録音停止
5. Whisper.cppで文字起こし
6. Ollamaで文脈補正
7. 補正後テキストをカーソル位置へ挿入
8. 成功通知（短時間表示）後、Idleへ戻る

### 3.2 エラーフロー
- マイク権限なし: 権限案内を表示し処理中断
- 文字起こし失敗: エラー通知し再試行導線を表示
- 補正失敗: 生文字起こし結果で挿入するかキャンセルを選択
- 挿入失敗: クリップボードへ結果保存し、手動貼り付けを案内

## 4. 機能仕様
### FD-01 メニューバー常駐
- アプリ起動時にメニューバーアイコンを表示する
- メニューから「設定」「終了」を選択できる

### FD-02 グローバルショートカット
- ユーザーが設定したキーで開始/停止をトグルできる
- 無効なショートカット設定時は保存を拒否する

### FD-03 録音モーダル
- 録音開始時に前面へ表示する
- 要素: 録音状態、経過時間、停止ボタン
- 録音停止またはエラー時に閉じる

### FD-04 音声収録
- 1セッション単位で音声バッファを保持する
- 停止時に一時音声ファイルへ保存し、後段処理へ引き渡す

### FD-05 文字起こし（Whisper.cpp）
- 音声ファイルを入力として文字起こしを実行する
- デフォルト言語は日本語（`ja`）として実行する
- 実行進行中は状態をTranscribingとして扱う
- 結果テキストまたは失敗理由を返す

### FD-06 文脈補正（Ollama）
- 入力: 文字起こし結果
- 出力: 補正後テキスト
- 補正ルール: 意味維持、誤字脱字・変換ミス優先修正、過度な要約禁止

### FD-07 テキスト挿入
- 第一手段: アクティブアプリのカーソル位置へ直接挿入
- 第二手段: クリップボードへ格納後、貼り付け操作を促す

### FD-08 設定
- 設定可能項目
- グローバルショートカット
- Whisperモデルパスまたはモデル種別
- Ollamaモデル名
- Ollama補正プロンプトテンプレート（`{{transcript}}` プレースホルダー対応）
- 補正ON/OFF

## 5. 例外・制約
- 連続セッション実行時、前回処理が完了するまで次セッション開始を制限する
- 無音が一定時間続いた場合の自動停止はMVPでは任意（初期は未実装可）
- 長時間録音は性能劣化を招くためMVPでは最大録音時間の上限設定を許容する

## 6. ログとユーザー通知
- ユーザー向け通知
- 開始、停止、成功、失敗を短いメッセージで表示
- 開発向けログ
- セッションID単位で「録音開始/停止」「文字起こし開始/終了」「補正開始/終了」「挿入結果」を記録

## 7. 受け入れ観点
- 開始から挿入完了まで、ショートカットと停止操作のみで完結する
- エラー時に次の行動（再試行、設定確認、手動貼り付け）が明確である
- 補正ON時、明らかな誤変換が軽減される
